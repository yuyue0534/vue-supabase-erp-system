# ERP 新人使用指南

> 适用范围：当前已完成的 Vue 3 + Supabase ERP 项目。  
> 覆盖模块：系统管理、基础数据、采购、销售、库存、财务、报表中心、仪表盘。  
> 本指南面向刚进入系统的管理员、业务员、采购员、仓库人员和财务人员。

---

## 1. 系统是怎样工作的

系统围绕一条标准经营链路运行：

```text
基础资料
→ 采购订单 → 收货入库 → 应付账款 → 付款
→ 销售订单 → 发货出库 → 应收账款 → 收款
→ 库存调整 / 盘点 / 调拨
→ 仪表盘与报表复盘
```

核心原则：

- **先建基础资料，再录业务单据。**
- **单据审核、过账后不应直接改数量或金额。** 发现错误时，应使用退货、红冲、反冲或新的调整单处理。
- **库存、应收、应付以已过账结果为准。**
- **权限决定能看什么、能新增什么、能审核什么。**

## 2. 登录、账号与角色

### 2.1 登录

打开前端地址后，使用管理员在 Supabase Authentication 中创建的邮箱和密码登录。

当前系统按“内部员工账号”模式运行，登录页没有公开注册入口。

### 2.2 新员工开通账号（无 Edge Function 模式）

当前项目不依赖 Edge Function 管理 Auth 用户，因此管理员按以下方式开通账号：

1. 进入 Supabase Dashboard。
2. 打开 **Authentication → Users**。
3. 点击 **Add user / Create new user**，填写员工邮箱与初始密码。
4. 在 SQL Editor 查询新用户 UUID：

```sql
select id, email, created_at
from auth.users
order by created_at desc;
```

5. 为该用户分配 ERP 角色并启用：

```sql
select public.erp_assign_role_to_auth_user(
  '替换为-auth.users-id'::uuid,
  'SALES',
  true
);
```

常用角色编码：

| 角色编码 | 适用人员 |
|---|---|
| `SUPER_ADMIN` | 系统最高管理员 |
| `ERP_MANAGER` | 业务管理员 |
| `PURCHASER` | 采购人员 |
| `SALES` | 销售人员 |
| `WAREHOUSE_KEEPER` | 仓库人员 |
| `FINANCE` | 财务人员 |
| `VIEWER` | 只读查看人员 |

### 2.3 停用员工 ERP 权限

不要直接删除历史业务数据。员工离职或暂停使用时，建议只停用 ERP 档案：

```sql
update public.sys_users
set status = 'inactive'
where auth_user_id = '替换为-auth.users-id'::uuid;
```

恢复时将 `status` 改回 `active`。

### 2.4 个人资料

右上角头像菜单进入 **个人资料**，可修改当前用户自己的用户名、真实姓名、手机号、头像资料；邮箱和角色为只读信息。

密码建议通过 Supabase 的安全流程重置，不要把服务端管理密钥写入前端。

## 3. 首次使用：推荐初始化顺序

首次上线或创建一套新账套时，按下面顺序建立数据。

### 3.1 系统管理

管理员先检查：

- 用户与角色是否正确；
- 权限是否满足岗位需求；
- 菜单是否符合业务范围；
- 操作日志是否可查询。

### 3.2 基础数据

建议按这个顺序维护：

1. **计量单位**：件、箱、公斤、米、套等。
2. **商品分类**：按产品线建立分类树。
3. **仓库与库位**：例如“主仓 → A区 → A-01-01”。
4. **供应商**：名称、编码、联系人、账期、评级等。
5. **客户**：名称、编码、联系人、账期、信用额度、客户级别。
6. **商品档案**：编码、名称、分类、单位、条码、价格、安全库存、图片。

只有在商品、仓库、客户、供应商都已经存在的前提下，采购和销售单据才能正常录入。

## 4. 基础数据模块操作说明

### 4.1 客户管理

路径：**基础数据 → 客户管理**

可维护：

- 客户编码、名称、客户级别；
- 联系人、电话、地址；
- 账期；
- 信用额度；
- 启用/停用状态。

注意：

- 客户编码应保持唯一且稳定，避免后期改码。
- 信用额度用于销售发货时的风险提示或控制。
- 客户停用后，不建议用于新建销售订单，但历史单据仍可保留查询。

### 4.2 供应商管理

路径：**基础数据 → 供应商管理**

可维护：

- 供应商编码、名称、联系人；
- 账期、评级、地址、状态；
- 采购相关备注。

建议给供应商设置合理账期，后续采购收货生成应付时会使用该信息推算到期日。

### 4.3 仓库与库位

路径：**基础数据 → 仓库管理**

建立仓库后，再建立库位。库位适合需要精确定位货物位置的场景。

建议：

- 小型业务可只建仓库，不细分库位。
- 货物种类较多时，按“区域—货架—层位”建立库位编码。
- 仓库的“允许负库存”应谨慎开启。生产环境通常建议关闭。

### 4.4 商品分类、商品和条码

路径：

- **基础数据 → 商品分类**
- **基础数据 → 商品档案**

商品档案建议至少填写：

- 商品编码；
- 商品名称；
- 分类；
- 主计量单位；
- 默认采购价、销售价；
- 安全库存；
- 条码；
- 是否启用。

商品图片上传后保存在 Supabase Storage，页面展示使用私有签名 URL，不会公开暴露原始文件。

批量导入商品时：

1. 先下载 Excel 模板；
2. 按模板字段填写；
3. 分类编码、单位编码必须已存在；
4. 导入前先检查商品编码是否重复；
5. 建议先用少量测试行验证，再导入全量数据。

## 5. 采购业务操作流程

### 5.1 推荐流程

```text
采购订单
→ 提交 / 审核
→ 采购收货
→ 审核并过账
→ 库存增加
→ 自动生成应付
→ 采购对账
→ 付款单
```

### 5.2 采购订单

路径：**采购管理 → 采购订单**

填写步骤：

1. 选择供应商。
2. 选择收货仓库。
3. 添加商品明细。
4. 填写数量、采购单价、税率、备注。
5. 保存草稿或提交审核。
6. 审核通过后，后续可按订单创建收货单。

系统支持键盘录入和商品选择弹窗。采购订单可支持部分收货。

### 5.3 采购收货

路径：**采购管理 → 采购收货**

操作要点：

1. 基于已审核采购订单创建收货单。
2. 填写本次实际到货数量、仓库、库位。
3. 允许分批收货。
4. 审核并过账后，系统会：
   - 增加库存；
   - 写入库存流水；
   - 更新采购订单已收数量；
   - 生成或更新应付账款。

### 5.4 采购退货

路径：**采购管理 → 采购退货**

适用于质量不合格、到货错误、采购取消等场景。

建议从原收货记录创建退货单，避免手工录入错误。退货过账后会减少库存；若已执行 P0 数据完整性加固迁移，还会形成供应商借项并按规则冲减相关应付。

### 5.5 采购对账

路径：**采购管理 → 采购对账**

按供应商和时间范围汇总采购金额、应付余额与付款情况，用于与供应商核对账目。

## 6. 销售业务操作流程

### 6.1 推荐流程

```text
销售订单
→ 信用额度 / 库存检查
→ 提交 / 审核
→ 销售发货
→ 审核并过账
→ 库存减少
→ 自动生成应收
→ 销售对账
→ 收款单
```

### 6.2 销售订单

路径：**销售管理 → 销售订单**

填写步骤：

1. 选择客户。
2. 选择商品和销售数量。
3. 填写单价、折扣、税率。
4. 根据需要切换含税/不含税。
5. 查看库存不足提示、客户信用额度提示。
6. 保存、提交、审核。

注意：前端的库存和信用提示用于辅助判断；最终是否允许过账应以数据库规则为准。

### 6.3 销售发货

路径：**销售管理 → 销售发货**

从销售订单创建发货单，可分批发货。

过账后系统会：

- 扣减指定仓库/库位库存；
- 写入库存流水；
- 更新订单已发货数量；
- 生成应收账款；
- 更新客户信用占用。

### 6.4 销售退货

路径：**销售管理 → 销售退货**

建议基于原发货记录创建，避免退货数量超过历史发货数量。

退货过账后会回补合格商品库存。若已执行 P0 数据完整性加固迁移，系统会形成客户贷项并处理原应收的冲减逻辑；已收款部分可能保留为客户贷项余额或进入后续退款流程。

### 6.5 销售对账

路径：**销售管理 → 销售对账**

用于查看客户销售金额、应收余额、收款情况及账龄。

## 7. 库存管理操作说明

### 7.1 库存查询

路径：**库存管理 → 库存查询**

可按商品、分类、仓库、库位查询。

重点关注：

- 现存数量；
- 可用库存；
- 单位成本、库存金额；
- 安全库存预警；
- 库龄信息。

### 7.2 库存调整

路径：**库存管理 → 库存调整**

适用于账面数量与实际数量不一致、损耗、盘盈等情况。

- 调盈：增加库存；
- 调亏：减少库存；
- 必须填写调整原因；
- 审核过账后生成库存流水。

库存调整不应替代正常的采购收货、销售发货、退货流程。

### 7.3 库存盘点

路径：**库存管理 → 库存盘点**

推荐步骤：

1. 新建盘点单，选择仓库或商品范围。
2. 生成盘点快照。
3. 录入实际数量。
4. 查看盘盈、盘亏差异。
5. 审核并过账，系统依据差异生成库存变动。

盘点期间建议减少同一仓库的频繁出入库，或明确盘点截止时点，避免账面数持续变化影响盘点结果。

### 7.4 库存调拨

路径：**库存管理 → 库存调拨**

用于仓库之间转移货物。

填写：

- 调出仓库/库位；
- 调入仓库/库位；
- 商品和数量；
- 调拨原因。

调拨过账后会生成调出和调入两类库存流水。若业务需要运输过程管理，应后续启用“在途库存”流程。

### 7.5 库存流水

路径：**库存管理 → 库存流水**

用于追溯每一笔入库、出库、调整、盘点、调拨记录。出现库存差异时，应先通过库存流水定位来源单据，而不是直接修改库存余额。

## 8. 财务管理操作说明

### 8.1 应收账款

路径：**财务管理 → 应收账款**

销售发货过账后会自动生成应收。

可查看：

- 客户；
- 原始应收金额；
- 已收金额；
- 未收余额；
- 到期日；
- 账龄和逾期状态。

### 8.2 应付账款

路径：**财务管理 → 应付账款**

采购收货过账后会自动生成应付。

可查看供应商、应付余额、到期日、已付金额和账龄。

### 8.3 收款单

路径：**财务管理 → 收款单**

用于客户回款和核销应收。

步骤：

1. 选择客户。
2. 选择收款方式：现金、银行转账、支票等。
3. 填写本次收款金额。
4. 勾选需要核销的应收单并分配金额。
5. 确保各明细核销金额之和不超过收款金额。
6. 审核并过账。

过账后，应收余额和客户信用占用会同步更新。

### 8.4 付款单

路径：**财务管理 → 付款单**

用于支付供应商货款并核销应付，流程与收款单相同。

### 8.5 费用单

路径：**财务管理 → 费用单**

用于记录非采购类支出，例如运费、办公费、差旅费、仓储费等。

填写费用类别、金额、税率、发生日期和备注；审核过账后进入费用统计与财务报表。

## 9. 仪表盘与报表

### 9.1 仪表盘

路径：**仪表盘**

通常包含：

- 今日销售额；
- 本月销售额；
- 待处理订单；
- 库存预警数量；
- 近 30 天销售趋势；
- 商品销售 TOP10；
- 库存预警列表；
- 待审核单据；
- 最近操作日志。

建议管理者每天先看仪表盘，再进入异常模块处理。

### 9.2 报表中心

路径：

- **报表中心 → 销售报表**
- **报表中心 → 采购报表**
- **报表中心 → 库存报表**
- **报表中心 → 财务报表**

报表支持按日期、客户、供应商、商品等条件筛选，并支持导出 Excel；图表可导出图片。

使用报表时请注意：

- 报表通常以已过账数据为核心；
- 日期范围过大时加载会较慢；
- 大规模数据场景应优先使用数据库聚合或专用报表视图。

## 10. 单据状态与操作边界

常见状态含义：

| 状态 | 含义 | 是否可继续编辑 |
|---|---|---|
| 草稿 | 未提交业务审批 | 可以 |
| 已提交 / 审核中 | 等待审批 | 通常不建议改；应退回后修改 |
| 已审核 | 可创建后续履约单据 | 视业务规则而定 |
| 部分收货 / 部分发货 | 已履约一部分 | 可以继续创建后续收货/发货 |
| 已完成 | 已全部履约 | 不应直接改 |
| 已过账 | 已影响库存或财务余额 | 不应直接改或删除 |
| 已关闭 / 已取消 | 业务结束 | 不应继续履约 |

原则：**一旦影响库存、应收、应付，就不要直接修改原单据。**

## 11. 每日、每周、每月建议操作

### 每日

- 处理待审核采购单、销售单、收货单、发货单；
- 查看库存预警；
- 检查当天销售发货和采购收货是否已过账；
- 财务人员核对当天收款、付款；
- 管理者查看仪表盘异常项。

### 每周

- 查看客户应收账龄和供应商应付账龄；
- 核查高频商品库存与安全库存；
- 审查库存调整和盘亏单；
- 处理长期未完成订单；
- 导出销售、采购、库存周报。

### 每月

- 做月末盘点或抽盘；
- 对客户、供应商进行对账；
- 检查未核销应收应付；
- 审核费用单和经营报表；
- 备份 Supabase 数据库或创建数据库分支；
- 汇总本月销售、采购、库存价值、回款和付款。

## 12. 常见问题

### 登录成功但看不到菜单

检查：

1. `sys_users.status` 是否为 `active`；
2. 是否已分配角色；
3. 角色是否有菜单和权限；
4. 退出并重新登录，刷新授权缓存。

### 新增单据时没有客户、供应商、商品或仓库可选

先检查对应基础资料是否已建立且状态为启用。

### 过账失败或提示库存不足

检查：

- 发货/退货/调整使用的仓库和库位是否正确；
- 当前库存是否足够；
- 仓库是否允许负库存；
- 单据是否已审核；
- 当前用户是否具有相应审核/过账权限。

### 收款或付款无法保存

检查：

- 收款/付款总金额是否大于明细核销金额之和；
- 每一行核销金额是否超过应收/应付剩余余额；
- 单据是否已被过账；
- 是否选错客户或供应商。

### 页面显示旧菜单或旧权限

退出登录后重新登录；仍异常时清除浏览器该站点缓存，再重新登录。

## 13. 数据安全与使用纪律

- 不要把 Supabase `service_role`、数据库密码写入前端 `.env` 或上传到 Git。
- 业务人员不应直接在 SQL Editor 修改库存余额、应收余额、应付余额。
- 真实数据上线前，应确认已执行数据库完整性加固迁移，例如已过账单据锁定、库存成本修复、信用控制、退货财务闭环等。
- 不要通过删除历史单据“修正错误”；应使用规范的退货、调整、红冲、反冲流程。
- 定期检查操作日志，尤其是库存调整、盘点、价格修改、权限修改。
- 新上线时先使用测试客户、测试供应商和测试仓库跑完整链路，再录入真实数据。

## 14. 新人快速上手清单

完成以下事项后，通常即可独立处理日常业务：

- [ ] 能登录并确认自己的角色。
- [ ] 会查询客户、供应商、商品、仓库和库存。
